# Phase 19 -> Phase 20 交接

## 当前完成进度

- 新增 `/diagnostics` 只读诊断页面：
  - 复用现有页面鉴权；启用 Web Token 时会跳转 `/auth`。
  - 调用 `/api/diagnostics` 获取脱敏状态。
  - 展示配置、访问、运行和目录四组状态。
  - 提供手动刷新按钮。
- 底部导航新增“诊断”入口，导航改为四列。
- 前端诊断渲染采用固定白名单：
  - 只显示布尔、枚举和数值状态。
  - 不显示 raw host、手机号、代理原文、代理用户名/密码、StringSession、`.session` 路径、Web Token、Cookie 或导出令牌。
  - 即使 API 响应中意外出现 secret 字段，页面也不会渲染这些字段。
- 纯 Node smoke 新增 diagnostics 分组，覆盖正常渲染、secret 注入不显示和接口失败状态。
- 后端页面 smoke 覆盖 `/diagnostics` 可渲染，Web Token 测试覆盖 `/diagnostics` 受保护。
- README、`docs/runtime-runbook.md` 和 `Telegram_Web开发.md` 已同步。
