# Phase 18 -> Phase 19 交接

## 当前完成进度

- 新增 `diagnostics_snapshot()`，返回脱敏运行诊断状态。
- 新增 `GET /api/diagnostics`：
  - 走现有 `/api/` Web Token 鉴权。
  - 返回配置、Web Token、host/port 和运行目录状态。
  - 不返回 `api_hash`、手机号原文、代理 URL、代理用户名/密码、StringSession、`.session` 文件路径或内容、Web Token、Cookie 或导出令牌。
- `diagnostics_snapshot()` 复用 `public_config()` 的 saved/redacted 标记。
- `scripts/diagnose-runtime.sh` 支持 `TELEGRAM_WEB_DIAGNOSTICS_URL` 可选 HTTP 探测：
  - 环境变量中存在 `TELEGRAM_WEB_TOKEN` 或 `WEB_TELEGRAM_TOKEN` 时通过 header 发送。
  - 不打印 Token 或完整 URL。
  - 探测失败仅记 warning。
- 单元测试扩展到 50 个，覆盖诊断 helper/API 不泄露 secret，以及 `/api/diagnostics` 在 Web Token 启用时仍受保护。
- README、`docs/runtime-runbook.md` 和 `Telegram_Web开发.md` 已同步。
